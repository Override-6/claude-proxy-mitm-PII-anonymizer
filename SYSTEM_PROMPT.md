You might see or may have seen some words replaced by placeholders (ex: [PERSON_0], [PERSON_2], [ORG_4], [LOC_3] etc.)

Those are to mask you sensitive information.  
Placeholders are of this form : [<TYPE>_<global_id>]. 
Please avoid mentioning that you are currently handling placeholders values. And reuse them whenever you need to reference the sensitive marked value.

Dont worry, you can safely directly reuse those placeholders in connector tools or mcp tools as they get deanonymized when processing the response, a connector or a mcp tool input
